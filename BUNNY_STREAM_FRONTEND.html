EduTrack Online - Bunny Stream API

Target Audience: Next.js Frontend Developers Library ID: 698152 CDN Hostname: vz-964b4290-bbe.b-cdn.net

Overview

All videos on EduTrack are hosted on Bunny Stream. The backend never stores video files — it relays uploads to Bunny and generates signed playback URLs.

Upload Flow

Teacher Browser → `POST /courses/videos/<id>/upload/` → Backend streams to Bunny CDN

Playback Flow

Student Browser → `GET /courses/videos/<id>/play/` → Backend checks permissions → Returns signed HLS URL
                                            ↘ Student streams directly from Bunny CDN (0 server load)

1. Video CRUD

POST /courses/videos/

Description: Create a video metadata record. bunny_video_id is null until the upload flow is completed.

Authentication: Teacher (must own the lecture's course)

Request Body:

{
  "lecture": "integer - Lecture ID",
  "name": "string - Video title",
  "order": "integer - Display order (optional, default: 0)",
  "is_active": "boolean (optional, default: true)"
}

Success Response — 201 Created:

{
  "id": "integer",
  "lecture": "integer",
  "name": "string",
  "bunny_video_id": null,
  "bunny_status": null,
  "bunny_status_display": "Pending",
  "order": "integer",
  "is_active": "boolean",
  "created_at": "datetime (ISO 8601)",
  "updated_at": "datetime (ISO 8601)"
}

GET /courses/videos/

Description: List videos. Supports filtering and ordering.

Authentication: Any authenticated user

Query Parameters:

Parameter Type Description
lecture Integer Filter by lecture ID
bunny_status Integer Filter by encoding status
is_active Boolean Filter by active status
search String Search by name
ordering String order, created_at

Success Response — 200 OK:

{
  "count": "integer",
  "results": [
    {
      "id": "integer",
      "lecture": "integer",
      "name": "string",
      "bunny_video_id": "string | null",
      "bunny_status": "integer | null",
      "bunny_status_display": "string",
      "order": "integer",
      "is_active": "boolean",
      "created_at": "datetime (ISO 8601)",
      "updated_at": "datetime (ISO 8601)"
    }
  ]
}

Status reference:

bunny_status bunny_status_display
null Pending
0 Created
1 Uploaded
2 Processing
3 Transcoding
4 Finished
5 Error

GET /courses/videos/<id>/

Description: Get a single video's details.

Authentication: Any authenticated user

Success Response — 200 OK: Same structure as list item.

PUT/PATCH /courses/videos/<id>/

Description: Update video metadata (name, order, is_active).

Authentication: Teacher (must own the lecture's course)

Request Body:

{
  "name": "string - New title",
  "order": "integer",
  "is_active": "boolean"
}

Success Response — 200 OK: Updated video object.

DELETE /courses/videos/<id>/

Description: Delete a video and remove it from Bunny Stream.

Authentication: Teacher (must own the lecture's course)

Success Response — 204 No Content

2. Upload Flow

POST /courses/videos/<id>/create-upload/

Description: Creates a video entry on Bunny Stream and returns the Bunny GUID. Must be called before uploading. The returned upload_url is for backend use only — the frontend should upload via the proxy endpoint instead.

Authentication: Teacher (must own the lecture's course)

Request Body: None

Success Response — 201 Created:

{
  "video_id": "integer - Local video ID",
  "bunny_video_id": "string - Bunny Stream GUID",
  "upload_url": "string - Bunny CDN upload URL (backend use only)",
  "status": "integer - Bunny status (0 = Created)"
}

POST /courses/videos/<id>/upload/

Description: Upload a video file to Bunny Stream via the backend proxy. The file is streamed chunk-by-chunk to Bunny — never stored on disk. Must call create-upload/ first.

Authentication: Teacher (must own the lecture's course)

Content-Type: multipart/form-data

Request Body:

Field Type Description
file File Video file (any format supported by Bunny Stream)

Success Response — 200 OK:

{
  "success": true,
  "video_id": "integer",
  "bunny_status": 1,
  "bunny_status_display": "Uploaded"
}

Error Responses:

Status Error Cause
400 "No Bunny video entry found" create-upload/ was not called first
400 "No file provided" Missing file field in form data
403 "You can only upload videos to your own courses" Wrong teacher
502 "Upload to Bunny Stream failed" Bunny CDN rejected the upload
500 "Upload failed due to an unexpected error" Internal error

3. Playback

GET /courses/videos/<id>/play/

Description: Get a signed HLS playback URL for a video. Multiple security checks are performed before the URL is returned.

Authentication: Student, Teacher, Assistant, SiteOwner

Access Checks (Student):

Check Blocked?
Video not found or not active 404
Student not authenticated 401
Lecture not purchased 403
Purchase expired 403
Watching limit reached (default: 4) 403
Prerequisite homework not passed 403
Prerequisite quiz not passed 403
Prerequisite exam not passed 403

Teacher/Assistant: Can preview any video in their own courses (skips purchase/prerequisite checks).

Success Response — 200 OK:

{
  "playback_url": "string - Signed HLS URL (expires in 2 hours)",
  "expires_at": "integer - Unix timestamp when the URL expires"
}

Error Responses:

Status Error Cause
404 "Video is not available." Video not found or inactive
401 "Authentication credentials were not provided." Not logged in
403 "You have not purchased this lecture." Student didn't buy
403 "Your access to this lecture has expired." Purchase expired
403 "You have reached the maximum watching limit for this lecture." Watch count exhausted
403 "You must pass the homework (ID X) before accessing this video." Prerequisite not met
403 "You can only preview videos in your own courses." Teacher trying to preview another teacher's video

4. Webhook

POST /webhooks/bunny-encoded/

Description: Called by Bunny Stream when video encoding is complete. Updates the video's bunny_status to 4 (Finished).

Authentication: HMAC signature verification (backend validates request authenticity)

Request Body (Bunny sends this):

{
  "VideoLibraryId": 698152,
  "VideoGuid": "string - Bunny Stream video GUID",
  "Status": "integer - New status"
}

Success Response — 200 OK:

{
  "status": "updated"
}

5. Watching Times Limit

Each lecture enforces a watching times limit (default: 4). Once a student has used all their watching times, the lecture's videos are locked.

Detail Value
Scope Per-student, per-lecture (watching any video counts)
Default 4
Max reopens 2 (each grants 1 extra day, does NOT reset watch count)
Locked access Homework, quizzes, materials still accessible
Teacher view watch_count and max_watch_count available in purchase details
