Bunny Stream — Frontend Integration Guide (Next.js)

Overview

All videos on EduTrack are hosted on Bunny Stream (Library ID: 698152). The upload and playback flow involves three actors:

Teacher's Browser → Our Backend (Django) → Bunny Stream CDN
                                       ↘ Student's Browser (playback)

Our backend never stores video files — it only relays them. Video playback comes directly from Bunny's CDN, 0 load on our server.

Architecture Summary

Step What happens Who does it
1 Teacher creates a video record (metadata only) Frontend → Backend API
2 Backend creates a video entry on Bunny Backend → Bunny API
3 Frontend uploads the file to the backend Frontend → Backend (multipart)
4 Backend streams the file to Bunny CDN Backend → Bunny CDN
5 Bunny encodes the video Bunny (automatic)
6 Bunny calls webhook when done Bunny → Backend
7 Frontend polls for status Frontend → Backend API
8 Student requests signed playback URL Frontend → Backend API
9 Student watches video from Bunny CDN Student browser → Bunny CDN

Step-by-Step API Flow (What the Frontend Must Do)

Step 1: Teacher Creates a Video Record

POST /courses/videos/

Request body:

{
    "lecture": 5,
    "name": "Chapter 1 - Intro",
    "order": 1
}

Response — 201: The video record is created. bunny_video_id is null, bunny_status_display is "Pending".

Frontend: Show the video in a table with status "Pending" and an "Upload" button.

Step 2: Teacher Clicks "Upload" → Generate Bunny Upload

POST /courses/videos/42/create-upload/

No request body needed. Backend creates a video entry on Bunny's servers.

Response — 201:

{
    "video_id": 42,
    "bunny_video_id": "abc-456-def-789",
    "upload_url": "https://video.bunnycdn.com/library/698152/videos/abc-456-def-789",
    "status": 0
}

The upload_url is for backend use only — never use it from the frontend directly. It requires the secret API key.

Frontend: After this call, update the UI state to "Ready to upload" (status bar moves from "Pending" to "Initialize").

Step 3: Upload the Video File (via Backend Proxy)

POST /courses/videos/42/upload/
Content-Type: multipart/form-data
Form field name: "file"

How to upload (Next.js):

Create an upload component that:

  1. Picks the file — <input type="file" accept="video/*" />
  2. Sends it as multipart/form-data — using FormData with the file appended as "file"
  3. Tracks progress — use XMLHttpRequest (not fetch) because XHR supports upload.onprogress for progress events. fetch does NOT support upload progress natively.
  4. Allows cancellation — call xhr.abort() to cancel

Why XHR and not fetch: The native fetch API does not provide upload progress events. You MUST use XMLHttpRequest for the upload to show progress. Create a small utility or use a library like axios which provides onUploadProgress.

// Upload utility example (Next.js compatible)

const uploadVideo = async (
  videoId: number,
  file: File,
  onProgress?: (percent: number) => void,
  signal?: AbortSignal
): Promise<{ success: boolean; bunny_video_id: string }> => {
  const formData = new FormData();
  formData.append('file', file);

  const xhr = new XMLHttpRequest();

  return new Promise((resolve, reject) => {
    xhr.upload.onprogress = (e) => {
      if (e.lengthComputable && onProgress) {
        onProgress(Math.round((e.loaded / e.total) * 100));
      }
    };

    xhr.onload = () => {
      if (xhr.status === 200) resolve(JSON.parse(xhr.responseText));
      else reject(new Error(JSON.parse(xhr.responseText).error || 'Upload failed'));
    };

    xhr.onerror = () => reject(new Error('Network error'));
    xhr.onabort = () => reject(new Error('Cancelled'));

    xhr.open('POST', `/courses/videos/${videoId}/upload/`);
    xhr.withCredentials = true;
    xhr.send(formData);

    if (signal) {
      signal.addEventListener('abort', () => xhr.abort());
    }
  });
};

Response — 200:

{
    "success": true,
    "video_id": 42,
    "bunny_status": 1,
    "bunny_status_display": "Uploaded"
}

Step 4: Poll for Encoding Status

After upload, poll GET /courses/videos/<id>/ every 5 seconds until bunny_status reaches 4.

Polling logic:

// Polling hook example (Next.js)

const useVideoStatus = (videoId: number) => {
  const [status, setStatus] = useState<number | null>(null);
  const [display, setDisplay] = useState('Pending');

  useEffect(() => {
    if (!videoId) return;

    const interval = setInterval(async () => {
      try {
        const res = await fetch(`/courses/videos/${videoId}/`, {
          credentials: 'include',
        });
        const data = await res.json();

        setStatus(data.bunny_status);
        setDisplay(data.bunny_status_display);

        if (data.bunny_status === 4 || data.bunny_status >= 5) {
          clearInterval(interval); // Stop polling
        }
      } catch {
        // Network error — keep polling
      }
    }, 5000);

    return () => clearInterval(interval);
  }, [videoId]);

  return { status, display };
};

Status values to display:

bunny_status Frontend Display
0 ⬜ Created — preparing upload
1 ⬜ Uploaded — encoding will start
2 🟡 Processing — analyzing file
3 🟡 Transcoding — encoding to resolutions
4 🟢 Ready — video playable
5 🔴 Error — encoding failed
6 🔴 Upload failed — try again

Step 5: Student Plays the Video

GET /courses/videos/42/play/

Backend checks before returning URL:

  1. Video exists and is active
  2. Student is authenticated
  3. Student purchased this lecture
  4. Purchase is not expired
  5. Watching limit not exhausted
  6. All prerequisites passed (if any)

Response — 200:

{
    "playback_url": "https://vz-964b4290-bbe.b-cdn.net/play/abc-456-def-789?token=abc123&expires=1720000000",
    "expires_at": 1720000000
}

Building the Video Player (Next.js)

1. Install hls.js:

npm install hls.js

2. Create a VideoPlayer component:

'use client';

import { useEffect, useRef, useState } from 'react';
import Hls from 'hls.js';

interface VideoPlayerProps {
  videoId: number;
  studentName?: string;
  studentCode?: string;
}

export default function VideoPlayer({ videoId, studentName, studentCode }: VideoPlayerProps) {
  const videoRef = useRef<HTMLVideoElement>(null);
  const containerRef = useRef<HTMLDivElement>(null);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    let hls: Hls | null = null;
    let tokenTimer: NodeJS.Timeout;

    const initPlayer = async () => {
      try {
        // 1. Fetch signed URL
        const res = await fetch(`/courses/videos/${videoId}/play/`, {
          credentials: 'include',
        });
        if (!res.ok) {
          const err = await res.json();
          throw new Error(err.error || 'Access denied');
        }
        const { playback_url, expires_at } = await res.json();

        if (!videoRef.current) return;

        // 2. Initialize HLS.js
        if (Hls.isSupported()) {
          hls = new Hls();
          hls.loadSource(playback_url);
          hls.attachMedia(videoRef.current);
        } else if (videoRef.current.canPlayType('application/vnd.apple.mpegurl')) {
          videoRef.current.src = playback_url;
        } else {
          throw new Error('HLS not supported in this browser');
        }

        // 3. Refresh token before expiry (at 90%)
        const expiresIn = expires_at * 1000 - Date.now();
        tokenTimer = setTimeout(refreshToken, expiresIn * 0.9);
      } catch (err: any) {
        setError(err.message);
      }
    };

    const refreshToken = async () => {
      try {
        const res = await fetch(`/courses/videos/${videoId}/play/`, {
          credentials: 'include',
        });
        const { playback_url, expires_at } = await res.json();

        if (hls) hls.loadSource(playback_url);
        else if (videoRef.current) videoRef.current.src = playback_url;

        const expiresIn = expires_at * 1000 - Date.now();
        tokenTimer = setTimeout(refreshToken, expiresIn * 0.9);
      } catch {}
    };

    initPlayer();
    return () => {
      hls?.destroy();
      clearTimeout(tokenTimer);
    };
  }, [videoId]);

  return (
    <div ref={containerRef} className="relative w-full aspect-video bg-black overflow-hidden">
      <video ref={videoRef} className="w-full h-full" controls />

      {/* Student ID overlay — anti-tampering measures applied via CSS */}
      {studentName && (
        <div className="student-overlay">
          {studentName} ({studentCode})
        </div>
      )}

      {error && (
        <div className="absolute inset-0 flex items-center justify-center text-white text-center p-5">
          {error}
        </div>
      )}
    </div>
  );
}

4. Track watch progress:

Create a hook that sends progress updates while the video plays:

const useWatchProgress = (videoId: number, videoRef: React.RefObject<HTMLVideoElement>) => {
  useEffect(() => {
    if (!videoId || !videoRef.current) return;

    let lastReported = 0;
    const interval = setInterval(async () => {
      const video = videoRef.current;
      if (!video || video.paused || video.ended) return;

      const currentTime = Math.floor(video.currentTime);
      const duration = Math.floor(video.duration);

      if (currentTime - lastReported >= 15) {
        lastReported = currentTime;
        try {
          await fetch('/courses/progress/update/', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            credentials: 'include',
            body: JSON.stringify({
              video: videoId,
              progress_seconds: currentTime,
              duration_seconds: duration || undefined,
            }),
          });
        } catch {}
      }
    }, 5000);

    return () => clearInterval(interval);
  }, [videoId, videoRef]);
};

Student ID Overlay — Anti-Tampering

The overlay displays the student's name and code on the video to deter sharing.

What the frontend should do:

  1. Use pointer-events: none — overlay can't be clicked (can't right-click → inspect on it)
  2. Use user-select: none — text can't be selected or highlighted
  3. Use CSS animation — overlay moves to random positions periodically, making it harder to block with a static CSS rule
  4. Re-create periodically — the React component re-renders on state changes, so the overlay element is recreated
  5. Position within the player container — overlay is inside the same overflow: hidden container as the video, so it can't be clipped out

What the student CANNOT do:

Attempt Result
Right-click → Inspect → Delete overlay Overlay reappears on re-render (React state cycle)
Use DevTools to hide .student-overlay CSS class can be randomized per session
Ad-block to block the element Overlay is inside the video container, not an external element
Download the video directly Bunny requires signed tokens — no permanent URL exists
Share the playback URL URL expires in 2 hours. New one requires purchase + auth check
Exceed watching limit Backend returns 403 after max_watch_count plays

The REAL security is NOT the overlay:

The overlay is a deterrent. The actual protection is:

  1. HMAC-signed playback URLs — student cannot forge a valid Bunny Stream token
  2. 2-hour token expiry — shared URLs stop working quickly
  3. Backend checks every play request — purchase, expiry, watch limit, prerequisites
  4. Watching limit enforcement — after max_watch_count (default 4), backend blocks new playback URLs
  5. No direct video URLs — every play requires GET /courses/videos/<id>/play/ which runs all security checks
