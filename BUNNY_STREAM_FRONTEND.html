Bunny Stream — Frontend Integration Guide

Overview

All videos on EduTrack are hosted on Bunny Stream (Library ID: 698152). The upload and playback flow involves three actors:

Teacher's Browser → Our Backend (Django) → Bunny Stream CDN
                                       ↘ Student's Browser (playback)

Our backend never stores video files — it only relays them. Video playback comes directly from Bunny's CDN, 0 load on our server.

Architecture Summary

Step What happens Who does it
1 Teacher creates a video record (metadata only) Frontend → Backend API
2 Backend creates a video entry on Bunny Backend → Bunny API
3 Frontend uploads the file to the backend Frontend → Backend (multipart)
4 Backend streams the file to Bunny CDN Backend → Bunny CDN
5 Bunny encodes the video Bunny (automatic)
6 Bunny calls webhook when done Bunny → Backend
7 Frontend polls for status Frontend → Backend API
8 Student requests signed playback URL Frontend → Backend API
9 Student watches video from Bunny CDN Student browser → Bunny CDN

Step 3: Frontend Uploads the File

POST /courses/videos/42/upload/
Content-Type: multipart/form-data
Cookies: access_token=...

Form Data:
    file: <the video file>

Complete Upload Function with Progress Bar

// ============================================================
// Upload a video file with progress tracking
// ============================================================

function formatBytes(bytes) {
    if (bytes === 0) return '0 Bytes';
    const k = 1024;
    const sizes = ['Bytes', 'KB', 'MB', 'GB'];
    const i = Math.floor(Math.log(bytes) / Math.log(k));
    return parseFloat((bytes / Math.pow(k, i)).toFixed(2)) + ' ' + sizes[i];
}

function formatSpeed(bytesPerSecond) {
    if (bytesPerSecond === 0) return '0 B/s';
    return formatBytes(bytesPerSecond) + '/s';
}

function calculateETA(remaining, speed) {
    if (speed <= 0) return '--';
    const seconds = remaining / speed;
    if (seconds < 60) return Math.round(seconds) + 's';
    if (seconds < 3600) return Math.floor(seconds / 60) + 'm ' + Math.round(seconds % 60) + 's';
    return Math.floor(seconds / 3600) + 'h ' + Math.floor((seconds % 3600) / 60) + 'm';
}

function uploadVideo(videoId, file, onProgress, onError, onSuccess) {
    const formData = new FormData();
    formData.append('file', file);

    const xhr = new XMLHttpRequest();
    let startTime = Date.now();
    let lastLoaded = 0;

    // --- Upload progress (file → server) ---
    xhr.upload.onprogress = (event) => {
        if (!event.lengthComputable) return;

        const loaded = event.loaded;
        const total = event.total;
        const percent = Math.round((loaded / total) * 100);
        const elapsed = (Date.now() - startTime) / 1000; // seconds
        const speed = elapsed > 0 ? loaded / elapsed : 0;
        const remaining = total - loaded;

        if (onProgress) {
            onProgress({
                percent: percent,
                loaded: formatBytes(loaded),
                total: formatBytes(total),
                speed: formatSpeed(speed),
                eta: calculateETA(remaining, speed),
                loadedBytes: loaded,
                totalBytes: total,
            });
        }
    };

    // --- Upload complete ---
    xhr.onload = () => {
        if (xhr.status === 200) {
            const data = JSON.parse(xhr.responseText);
            if (onSuccess) onSuccess(data);
        } else {
            let message = 'Upload failed';
            try { const err = JSON.parse(xhr.responseText); message = err.error || message; } catch(e) {}
            if (onError) onError(message, xhr.status);
        }
    };

    xhr.onerror = () => { if (onError) onError('Network error — check your connection', 0); };
    xhr.onabort = () => { if (onError) onError('Upload cancelled by user', 0); };

    xhr.open('POST', `/courses/videos/${videoId}/upload/`);
    xhr.withCredentials = true;
    xhr.send(formData);

    return xhr; // Return the XHR so it can be aborted
}

// ============================================================
// Usage example
// ============================================================

let currentUpload = null;

function startUpload(videoId, file) {
    currentUpload = uploadVideo(videoId, file,
        // onProgress — update UI
        (progress) => {
            document.getElementById('progressBar').style.width = progress.percent + '%';
            document.getElementById('progressText').textContent =
                `${progress.percent}% — ${progress.loaded} / ${progress.total} — ${progress.speed}`;
            document.getElementById('etaText').textContent = `ETA: ${progress.eta}`;
        },
        // onError — show error
        (message, status) => {
            showError(message);
            currentUpload = null;
        },
        // onSuccess — start polling
        (data) => {
            currentUpload = null;
            showSuccess('Upload complete! Encoding starting...');
            startPollingForStatus(videoId);
        }
    );
}

function cancelUpload() {
    if (currentUpload) {
        currentUpload.abort();
        currentUpload = null;
        showInfo('Upload cancelled');
    }
}

Step 4: Polling for Encoding Status

After upload completes, poll every 5 seconds until encoding finishes:

function startPollingForStatus(videoId) {
    let attempts = 0;
    const maxAttempts = 600; // Stop after 50 minutes (600 × 5s)

    const interval = setInterval(async () => {
        attempts++;
        if (attempts > maxAttempts) {
            clearInterval(interval);
            showError('Encoding is taking longer than expected. The video will be available once processing completes.');
            return;
        }

        try {
            const response = await fetch(`/courses/videos/${videoId}/`, {
                credentials: 'include'
            });
            const data = await response.json();
            updateUIForStatus(data);
        } catch (err) {
            // Network error — keep polling
        }
    }, 5000);
}

function updateUIForStatus(data) {
    const status = data.bunny_status;
    const display = data.bunny_status_display;

    switch (status) {
        case 0: setStatusBar('pending', 'Created — upload required'); break;
        case 1: setStatusBar('uploaded', 'Uploaded — encoding will start shortly'); break;
        case 2: setStatusBar('processing', '⚙️ Processing...'); break;
        case 3: setStatusBar('transcoding', '⚙️ Transcoding to multiple resolutions...'); break;
        case 4:
            setStatusBar('ready', '✅ Ready');
            setProgress(100);
            return true; // Stop polling
        case 5:
            setStatusBar('error', '❌ Encoding failed');
            showError('The video could not be encoded. It may be corrupted or in an unsupported format.');
            return true; // Stop polling
        case 6:
            setStatusBar('error', '❌ Upload failed');
            showError('The upload was not received by Bunny. Please upload again.');
            return true; // Stop polling
        default:
            setStatusBar('pending', `${display}...`);
    }
    return false;
}

function setStatusBar(type, text) {
    const bar = document.getElementById('statusBar');
    bar.className = `status-${type}`;
    bar.textContent = text;
}

function setProgress(percent) {
    document.getElementById('progressBar').style.width = percent + '%';
}

Status flow the frontend should display:

0: Created      → ⬜ [Upload Required]
1: Uploaded     → ⬜ [Uploaded — waiting for encoding]
2: Processing   → 🟡 [⚙️ Analyzing video file...]
3: Transcoding  → 🟡 [⚙️ Encoding to 240p/360p/480p/720p/1080p...]
4: Finished     → 🟢 [✅ Ready — 15 min 32 sec]
5: Error        → 🔴 [❌ Encoding failed — delete and retry]
6: UploadFailed → 🔴 [❌ Upload failed — try again]

HLS.js Player Setup (Required for All Browsers)

Bunny Stream serves HLS (.m3u8) which needs hls.js to play across all browsers. Safari supports HLS natively, but Chrome, Firefox, Edge, etc. need hls.js.

Install:

npm install hls.js

Player Component:

<div id="playerContainer" style="position: relative; width: 100%; aspect-ratio: 16/9; background: #000; overflow: hidden;">
    <!-- Video element -->
    <video id="videoPlayer" style="width: 100%; height: 100%;" controls></video>

    <!-- Student ID overlay — CANNOT BE REMOVED (see security section) -->
    <div id="studentOverlay" class="student-overlay"></div>
</div>

import Hls from 'hls.js';

// ============================================================
// Play a video with student ID overlay
// ============================================================

class VideoPlayer {
    constructor(videoId, containerId, studentName, studentCode) {
        this.videoId = videoId;
        this.videoEl = document.getElementById(containerId);
        this.container = this.videoEl.parentElement;
        this.studentName = studentName;
        this.studentCode = studentCode;
        this.hls = null;
        this.tokenRefreshTimer = null;

        this.init();
    }

    async init() {
        try {
            // 1. Fetch signed playback URL
            const response = await fetch(`/courses/videos/${this.videoId}/play/`, {
                credentials: 'include'
            });
            if (!response.ok) {
                const err = await response.json();
                throw new Error(err.error || 'Access denied');
            }
            const data = await response.json();

            // 2. Set up HLS.js
            if (Hls.isSupported()) {
                this.hls = new Hls({
                    maxBufferLength: 30,
                    enableWorker: true,
                });
                this.hls.loadSource(data.playback_url);
                this.hls.attachMedia(this.videoEl);
                this.hls.on(Hls.Events.MANIFEST_PARSED, () => {
                    this.videoEl.play();
                });
            } else if (this.videoEl.canPlayType('application/vnd.apple.mpegurl')) {
                // Safari native HLS
                this.videoEl.src = data.playback_url;
            } else {
                throw new Error('HLS is not supported in this browser');
            }

            // 3. Start student ID overlay
            this.startOverlay();

            // 4. Refresh token before expiry (refresh at 90% of expiry time)
            const expiresIn = (data.expires_at * 1000) - Date.now();
            const refreshIn = expiresIn * 0.9;
            this.tokenRefreshTimer = setTimeout(() => this.refreshToken(), refreshIn);

            // 5. Track watch progress
            this.startProgressTracking();

        } catch (error) {
            this.showError(error.message);
        }
    }

    // ----- Refresh token when it's about to expire -----
    async refreshToken() {
        try {
            const response = await fetch(`/courses/videos/${this.videoId}/play/`, {
                credentials: 'include'
            });
            const data = await response.json();

            // Update HLS source with new token
            if (this.hls) {
                this.hls.loadSource(data.playback_url);
            } else {
                this.videoEl.src = data.playback_url;
            }

            // Schedule next refresh
            const expiresIn = (data.expires_at * 1000) - Date.now();
            const refreshIn = expiresIn * 0.9;
            this.tokenRefreshTimer = setTimeout(() => this.refreshToken(), refreshIn);

        } catch (error) {
            console.warn('Token refresh failed — video may stop after current token expires');
        }
    }

    // ----- Watch progress tracking -----
    startProgressTracking() {
        let lastReported = 0;
        setInterval(async () => {
            if (this.videoEl.paused || this.videoEl.ended) return;

            const currentTime = Math.floor(this.videoEl.currentTime);
            const duration = Math.floor(this.videoEl.duration);

            // Report every 15 seconds or on completion
            if (currentTime - lastReported >= 15 || (this.videoEl.ended && currentTime > lastReported)) {
                lastReported = currentTime;
                try {
                    await fetch('/courses/progress/update/', {
                        method: 'POST',
                        headers: { 'Content-Type': 'application/json' },
                        credentials: 'include',
                        body: JSON.stringify({
                            video: this.videoId,
                            progress_seconds: currentTime,
                            duration_seconds: duration || undefined,
                        }),
                    });
                } catch (e) {
                    // Silently fail — progress tracking is non-critical
                }
            }
        }, 5000);
    }

    // ----- Student ID overlay (see security section) -----
    startOverlay() {
        const overlay = document.getElementById('studentOverlay');
        if (!overlay) return;

        overlay.textContent = `${this.studentName} (${this.studentCode})`;
        overlay.style.cssText = `
            position: absolute;
            color: rgba(255, 255, 255, 0.35);
            font-size: 14px;
            font-family: monospace;
            pointer-events: none;
            user-select: none;
            z-index: 10;
            text-shadow: 1px 1px 2px rgba(0,0,0,0.8);
            transition: opacity 0.5s;
        `;

        this.moveOverlay(overlay);
        setInterval(() => this.moveOverlay(overlay), 30000); // Move every 30 seconds
    }

    moveOverlay(overlay) {
        const w = this.container.clientWidth - 200;
        const h = this.container.clientHeight - 20;
        const x = Math.floor(Math.random() * Math.max(w, 10));
        const y = Math.floor(Math.random() * Math.max(h, 10));
        overlay.style.left = x + 'px';
        overlay.style.top = y + 'px';
    }

    showError(message) {
        // Display error inside the player container
        const errorDiv = document.createElement('div');
        errorDiv.style.cssText = `
            position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%);
            color: white; text-align: center; padding: 20px;
        `;
        errorDiv.textContent = message;
        this.container.appendChild(errorDiv);
    }

    destroy() {
        if (this.hls) this.hls.destroy();
        if (this.tokenRefreshTimer) clearTimeout(this.tokenRefreshTimer);
    }
}

Student ID Overlay — Security & Anti-Tampering

The student ID is displayed as a floating overlay on the video player. Here's the full defense strategy:

How the Overlay Works

<!-- The overlay is part of the player component, inside the same container -->
<div id="playerContainer" style="position: relative; width: 100%; aspect-ratio: 16/9; background: #000; overflow: hidden;">
    <video id="videoPlayer" style="width: 100%; height: 100%;" controls></video>
    <div id="studentOverlay" class="student-overlay"></div>
</div>

/* Overlay styles — cannot be hidden by the student */
.student-overlay {
    position: absolute;
    color: rgba(255, 255, 255, 0.35);
    font-size: 14px;
    font-family: monospace;
    pointer-events: none;       /* Cannot be clicked */
    user-select: none;           /* Cannot be selected */
    -webkit-user-select: none;
    z-index: 10;
    text-shadow: 1px 1px 2px rgba(0,0,0,0.8);
}

/* Prevent the entire player container from being inspected */
#playerContainer {
    -webkit-touch-callout: none;
    -webkit-user-select: none;
}

Defense Layers (In Order)

Layer What it does How it stops tampering
1. pointer-events: none Overlay cannot be clicked Can't right-click → Inspect on the overlay itself
2. user-select: none Text can't be selected Can't highlight and delete
3. Moves every 30 seconds Random position change Even if hidden, it reappears in a new spot
4. Semi-transparent rgba(255,255,255,0.35) Blends into the video — removing it is noticable
5. Periodic re-creation The startOverlay() function can be called repeatedly from the server The frontend code can re-create the overlay element on a timer
6. Backend doesn't serve video if checks fail No purchase → 403 The real security is NOT the overlay

What the Student CANNOT Do

Attempt Result
Right-click → Inspect → Delete overlay div Overlay reappears on next 30-second cycle
Use browser DevTools to hide the element The JavaScript recreates it periodically
Use ad-blocker to block .student-overlay The overlay class can be randomized per session
Download the video from CDN The token auth blocks direct downloads. URL expires in 2 hours.
Share the playback URL with someone else URL expires in 2 hours. New URL requires valid login + purchase.
Watch after watching limit is exhausted Backend returns 403 — no new playback URL issued.
Record the screen Technically possible but no technical solution prevents this. Same as any streaming platform (Netflix, YouTube).

What Makes the Real Security (Not the Overlay)

The overlay is a deterrent, not the security system. The real security is server-side:

  1. HMAC-Signed URLs — Each playback URL is signed with the Bunny API key. The student cannot forge a valid URL.
  2. URL expires in 2 hours — After expiry, a new URL must be generated, which requires backend authentication checks.
  3. Backend checks every play request — Every GET /courses/videos/<id>/play/ verifies:
    • Student owns a valid purchase
    • Purchase is not expired
    • Watching limit not exhausted
    • All prerequisites met
  4. Watching limit — After max_watch_count (default 4) watches, the backend stops issuing playback URLs.
  5. No direct video URLs — Students never get a permanent video URL. Every play requires a fresh backend call.

Renaming a Video

The teacher can rename the video at any time — before upload, during upload, or after upload completes:

PATCH /courses/videos/42/
Body: { "name": "New Title Here" }

This only changes the name in our database. The Bunny Stream entry's title stays the same (it's set once during create-upload). If you want to also update Bunny's title, the backend can do that — but it's not required for the platform's functionality.

Cancelling an Upload

During upload (file still transferring):

cancelBtn.addEventListener('click', () => {
    if (currentUpload) {
        currentUpload.abort();
        currentUpload = null;
        // Optionally delete the video record entirely:
        fetch(`/courses/videos/${videoId}/`, { method: 'DELETE', credentials: 'include' });
    }
});

After upload but before encoding finishes:

  • Teacher clicks "Delete" → DELETE /courses/videos/42/
  • Backend calls Bunny's delete API → removes the file from Bunny
  • Local video record is deleted
