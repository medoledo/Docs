GET /accounts/teachers//

Authentication: SiteOwner

Success Response — 200 OK: Same structure as list item.

POST /accounts/teachers/

Description: Create a new teacher user and profile atomically. Supports profile picture upload via multipart/form-data.

Authentication: SiteOwner

Content-Type: multipart/form-data

Request Body:

{
  "username": "String (Required)",
  "password": "String (Required) — Minimum 8 characters",
  "name": "String (Required)",
  "phone": "String (Required) — Egyptian format: 010/011/012/015 + 8 digits",
  "secondary_phone": "String (Optional) — Egyptian format",
  "gmail": "String (Required) — Must be globally unique",
  "gender": "String (Optional) — male|female",
  "profile_picture": "File (Optional) — image/jpeg|image/png|image/webp",
  "biography": "String (Optional)",
  "facebook": "String (Optional)",
  "subject": "Integer (Required) — Subject ID",
  "grades": "Array[Integer] (Required) — Grade IDs",
  "video_security": "Integer (Optional) — VideoSecurity ID"
}

Success Response — 201 Created: Same structure as GET list item.

Error Responses:

Status Condition Response Body
400 Username exists {"username": ["Username already exists."]}
400 Email exists {"gmail": ["This email is already associated with an account."]}
400 Invalid phone {"phone": ["Phone number must be exactly 11 digits..."]}
400 Weak password {"password": ["Password validation error"]}
403 Not authorized {"detail": "You do not have permission..."}

Business Rules:
- Username and email (gmail) must be globally unique across ALL user types.
- Phone numbers must be valid Egyptian numbers (010, 011, 012, or 015 prefix).
- A User account and TeacherProfile are created atomically in a transaction.
- subject is a single Subject ID.

PATCH /accounts/teachers//

Authentication: SiteOwner

Content-Type: multipart/form-data

Request Body: Same as POST (all fields optional for PATCH).

Success Response — 200 OK: Same structure as list item.

Error Responses:

Status Condition Response Body
400 Email exists (other user) {"gmail": ["This email is already associated with an account."]}
403 Not authorized {"detail": "You do not have permission..."}
404 Teacher not found {"detail": "Not found."}

{

3. Course Management

POST /courses/

Description: Create a new course.

Authentication: SiteOwner only

Content-Type: multipart/form-data

Request Body:

{
  "name": "String (Required)",
  "teacher": "Integer (Required) — Teacher ID",
  "grade": "Integer (Required) — Grade ID",
  "description": "String (Optional)",
  "cover_picture": "File (Optional) — image/jpeg|image/png|image/webp",
  "is_active": "Boolean (Optional) — Default: true"
}

Success Response — 201 Created:

{
  "id": "Integer",
  "name": "String",
  "teacher": "Integer",
  "teacher_id": "Integer",
  "teacher_name": "String",
  "grade": "Integer",
  "grade_name": "String",
  "subject": "Integer",
  "subject_name": "String",
  "description": "String | null",
  "cover_picture": "String (URL) | null",
  "is_active": "Boolean",
  "topics": "Array[Object]",
  "topic_count": "Integer",
  "total_lectures": "Integer",
  "total_videos": "Integer",
  "created_at": "DateTime (ISO 8601)",
  "updated_at": "DateTime (ISO 8601)"
}

Error Responses:

Status Condition Response Body
400 Teacher doesn't teach grade {"grade": ["Teacher X does not teach grade Y."]}
400 Duplicate course {"non_field_errors": ["A course with this Teacher, Grade, and Name already exists."]}
403 Not site owner {"detail": "You do not have permission..."}

Business Rules:
- The teacher must teach the selected grade.
- Subject is derived automatically from teacher.subject — it is NOT sent in the request.
- A teacher cannot have duplicate courses for the same (grade, name) combination.
- Teacher, grade, and name cannot be changed after course creation.

PATCH /courses//

Authentication: SiteOwner only

Content-Type: multipart/form-data

Request Body: Same as POST (all fields optional for PATCH).

Error Responses:

Status Condition Response Body
400 Changing teacher {"teacher": ["Cannot change the teacher of an existing course."]}
400 Changing grade {"grade": ["Cannot change the grade of an existing course."]}

Business Rules:
- Teacher and grade cannot be changed after course creation.
- Subject is derived from the teacher and updates automatically if the teacher's subject changes.

DELETE /courses//

Authentication: SiteOwner only

Error Responses:

Status Condition Response Body
400 Students enrolled {"detail": "Cannot delete a course that has students enrolled in it."}

Business Rules:
- Course can only be deleted if no students are enrolled (no approved enrollments exist).

5. Recharge Codes

POST /payments/codes/

Description: Generate new recharge codes. Codes are course-specific and can only be redeemed for the course they are generated for.

Authentication: SiteOwner

Content-Type: application/json

Request Body:

{
  "course": "Integer (Required) — Course ID",
  "value": "Decimal (Required) — Amount in EGP (minimum 1.00)",
  "expires_at": "DateTime (Optional) — ISO 8601, code cannot be redeemed after this date"
}

Success Response — 201 Created:

{
  "id": "Integer",
  "course": "Integer",
  "course_name": "String",
  "teacher_name": "String",
  "code": "String — Auto-generated (e.g. X7K9-M2P4-QR1W-L5D8)",
  "value": "Decimal",
  "is_used": false,
  "used_by": null,
  "used_by_name": null,
  "used_at": null,
  "created_at": "DateTime (ISO 8601)",
  "expires_at": "DateTime (ISO 8601) | null",
  "created_by": "Integer",
  "created_by_username": "String"
}

Error Responses:

Status Condition Response Body
400 Invalid course {"course": ["Invalid pk \"X\" - object does not exist."]}
400 Invalid value {"value": ["Ensure this value is greater than or equal to 1."]}
403 Not site owner {"detail": "You do not have permission..."}

Business Rules:
- Code format: XXXX-XXXX-XXXX-XXXX (16 random characters, cryptographically secure).
- Codes are one-time use only.
- Codes are course-specific — a code generated for Course A cannot be used for Course B.

GET /payments/revenue/trends/

Description: Revenue trends over time. Defaults to the past 30 days. Supports custom date ranges.

Authentication: SiteOwner

Query Parameters:

Parameter Type Description
start_date String (YYYY-MM-DD) Start of date range
end_date String (YYYY-MM-DD) End of date range

Success Response — 200 OK:

{
  "start_date": "2026-03-27",
  "end_date": "2026-04-26",
  "total_revenue": "String — Decimal as string",
  "total_purchases": "Integer",
  "daily_trends": [
    {
      "date": "String (YYYY-MM-DD)",
      "revenue": "String — Decimal as string",
      "purchase_count": "Integer"
    }
  ]
}

Business Rules:
- Default range: last 30 days if no dates provided.
- If only end_date provided, start_date = end_date - 30 days.
- If only start_date provided, end_date = today.
