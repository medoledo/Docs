POST /accounts/teachers/

Description: Create a new teacher user and profile atomically. Supports profile picture upload via multipart/form-data.

Authentication: SiteOwner

Content-Type: multipart/form-data

Request Body:

{
  "username": "String (Required)",
  "password": "String (Required) — Minimum 8 characters",
  "name": "String (Required)",
  "phone": "String (Required) — Egyptian format: 010/011/012/015 + 8 digits",
  "secondary_phone": "String (Optional) — Egyptian format",
  "gmail": "String (Required) — Must be globally unique",
  "gender": "String (Required) — male|female",
  "profile_picture": "File (Optional) — image/jpeg|image/png|image/webp",
  "biography": "String (Optional)",
  "facebook": "String (Optional)",
  "is_active": "Boolean (Optional) — Default: true",
  "subject": "Integer (Required) — Subject ID",
  "grades": "Array[Integer] (Required) — Grade IDs",
  "video_security": "Integer (Optional) — VideoSecurity ID"
}

Success Response — 201 Created: Returns the full teacher profile (same structure as GET detail above, including bunny_collection_id which will be null until assigned).

Error Responses:

Status Condition Response Body
400 Username exists {"username": ["Username already exists."]}
400 Email exists {"gmail": ["This email is already associated with an account."]}
400 Invalid phone {"phone": ["Phone number must be exactly 11 digits..."]}
400 Weak password {"password": ["Password validation error"]}
403 Not authorized {"detail": "You do not have permission..."}

Business Rules:

  • Username and email (gmail) must be globally unique across ALL user types.
  • Phone numbers must be valid Egyptian numbers (010, 011, 012, or 015 prefix).
  • A User account and TeacherProfile are created atomically in a transaction.
  • subject is a single Subject ID.
  • grades must be a subset of subject's available grades. For example, if a subject is only assigned to "3rd Secondary", the teacher cannot be assigned "1st Secondary".

PATCH /accounts/teachers/<id>/

Authentication: SiteOwner

Content-Type: multipart/form-data

Request Body: Same as POST body (all fields optional for PATCH). Additionally:

  • bunny_collection_id can be set directly here, or use the dedicated teacher update (set bunny_collection_id) endpoint below.
  • password can be set to reset the teacher's password directly (optional). The teacher can also use the forgot-password flow themselves.

Success Response — 200 OK: Returns the full teacher profile (same structure as GET detail above, including bunny_collection_id).

Error Responses:

Status Condition Response Body
400 Email exists (other user) {"gmail": ["This email is already associated with an account."]}
403 Not authorized {"detail": "You do not have permission..."}
404 Teacher not found {"detail": "Not found."}

Business Rules:

  • If a grade is removed from the teacher's profile, all courses for that teacher + grade will be set to is_active = False. The courses become hidden from both the teacher and enrolled students.
  • If the teacher's subject is changed, all active courses for that teacher will be set to is_active = False. The existing grades must also be valid for the new subject — adjust them in the same request if needed.

Teacher Collection Management

Teachers must have a Bunny Stream collection assigned before they can upload videos. Create a collection manually in your Bunny Stream dashboard → Collections → Create Collection, then assign it via the API.

The response for teacher list/detail views now includes bunny_collection_id.

PATCH /accounts/teachers/<id>/ (set bunny_collection_id directly)

Note: Collection assignment is done directly via PATCH /accounts/teachers/<id>/ — set bunny_collection_id in the update body. The dedicated teacher update (set bunny_collection_id)/ endpoint has been removed.

Authentication: SiteOwner

Content-Type: application/json

Request Body:

{
  "collection_id": "String (Required) — Bunny Stream Collection GUID"
}

Success Response — 200 OK:

{
  "bunny_collection_id": "String — The assigned collection GUID",
  "message": "Collection assigned to teacher."
}

Error Responses:

Status Condition Response Body
400 Missing collection_id {"error": "collection_id is required."}
404 Teacher not found {"error": "Teacher not found"}

PATCH /accounts/students/<id>/

Description: Update a student's profile fields and/or status. Status changes are logged with the acting user and reason.

Authentication: SiteOwner

Content-Type: application/json

Request Body:

{
  "name_ar": "String (Optional)",
  "name_en": "String (Optional)",
  "phone_number": "String (Optional)",
  "father_number": "String (Optional)",
  "mother_number": "String (Optional)",
  "school_type": "Integer (Optional)",
  "grade": "Integer (Optional)",
  "division": "Integer (Optional)",
  "school_name": "String (Optional)",
  "birth_date": "Date (YYYY-MM-DD, Optional)",
  "gender": "String (Optional)",
  "gmail": "String (Optional)",
  "governorate": "Integer (Optional)",
  "area": "Integer (Optional)",
  "status": "String (Optional) — verified|declined|suspended_temporary|suspended_permanent",
  "is_active": "Boolean (Optional)",
  "reason": "String (Optional) — Required for decline and suspension"
}

Allowed Status Transitions:

Current Status Allowed New Status
pending verified, declined
verified suspended_temporary, suspended_permanent
declined verified
suspended_temporary verified
suspended_permanent verified

Business Rules:

  • A reason is required when declining or suspending a student.
  • Changing to declined or suspended automatically sets is_active = false.
  • Changing to verified automatically sets is_active = true.
  • Cannot bring a student back to pending once they've been processed (only declined students can edit their own profile to reset to pending).
  • Every status change is recorded in status_history array on the student detail response.

Error Responses:

Status Condition Response Body
400 Transition not allowed {"status": ["Cannot change status from 'pending' to 'suspended_temporary'."]}
400 Missing reason for decline/suspend {"reason": ["A reason is required when declining or suspending a student."]}

  "cover_picture": "String (URL) | null",
  "is_active": "Boolean",
  "topics": "Array[Object]",
  "topic_count": "Integer",
  "total_lectures": "Integer",
  "total_videos": "Integer",
  "enrolled_count": "Integer | null - Number of approved students (teachers only)",
  "pending_count": "Integer | null - Number of pending requests (teachers only)",
  "rejected_count": "Integer | null - Number of rejected requests (teachers only)",
  "created_at": "DateTime (ISO 8601)",
  "updated_at": "DateTime (ISO 8601)"
}

Error Responses:

Status Condition Response Body
400 Teacher doesn't teach grade {"grade": ["Teacher X does not teach grade Y."]}
400 Subject not available for grade {"grade": ["Subject X is not available for grade Y."]}
400 Duplicate course {"non_field_errors": ["A course with this Teacher, Grade, and Name already exists."]}
403 Not site owner {"detail": "You do not have permission..."}

Business Rules:

  • The teacher must teach the selected grade.
  • Subject is derived automatically from teacher.subject — it is NOT sent in the request.
  • A teacher cannot have duplicate courses for the same (grade, name) combination.
  • Teacher, grade, and name cannot be changed after course creation.

PATCH /courses/<id>/

Authentication: SiteOwner only

Content-Type: multipart/form-data

Request Body: Same as POST (all fields optional for PATCH).

Error Responses:

Status Condition Response Body
400 Changing teacher {"teacher": ["Cannot change the teacher of an existing course."]}
400 Changing grade {"grade": ["Cannot change the grade of an existing course."]}

Business Rules:

  • Teacher and grade cannot be changed after course creation.
  • Subject is derived from the teacher and updates automatically if the teacher's subject changes.

DELETE /courses/<id>/

Authentication: SiteOwner only

Error Responses:

Status Condition Response Body
400 Students enrolled {"detail": "Cannot delete a course that has students enrolled in it."}

Business Rules:

  • Course can only be deleted if no students are enrolled (no approved enrollments exist).

5. Recharge Codes

POST /payments/codes/

Description: Generate new recharge codes. Codes are course-specific and can only be redeemed for the course they are generated for.

Authentication: SiteOwner, Teacher

Content-Type: application/json

Request Body:

{
  "course": "Integer (Required) — Course ID",
  "value": "Decimal (Required) — Amount in EGP (1.00 - 1000.00)",
  "expires_at": "DateTime (Optional) — ISO 8601, code cannot be redeemed after this date"
}

Success Response — 201 Created:

{
  "id": "Integer",
  "course": "Integer",
  "course_name": "String",
  "teacher_name": "String",
  "code": "String — Auto-generated (e.g. X7K9-M2P4-QR1W-L5D8)",
  "value": "Decimal",
  "status": "String (valid)",
  "status_display": "String (Valid)",
  "used_by": null,
  "used_by_name": null,
  "used_at": null,
  "created_at": "DateTime (ISO 8601)",
  "expires_at": "DateTime (ISO 8601) | null",
  "created_by": "Integer",
  "created_by_username": "String"
}

Error Responses:

Status Condition Response Body
400 Invalid course {"course": ["Invalid pk \"X\" - object does not exist."]}
400 Invalid value {"value": ["Ensure this value is greater than or equal to 1."]}
403 Not site owner {"detail": "You do not have permission..."}

Business Rules:

  • Code format: XXXX-XXXX-XXXX-XXXX (16 random characters, cryptographically secure).
  • Codes are one-time use only.
  • Codes are course-specific — a code generated for Course A cannot be used for Course B.

POST /payments/codes//request-blacklist-otp/

Description: Request an OTP to authorize blacklisting a recharge code. Sends a 6-digit OTP to the SiteOwner's registered gmail. The OTP expires in 30 minutes.

Authentication: SiteOwner

Success Response — 200 OK:

{
  "detail": "OTP sent to your email.",
  "email": "siteowner@gmail.com"
}

Error Responses:

Status Condition
400 Code is already used or already blacklisted

POST /payments/codes//blacklist/

Description: Permanently blacklist a recharge code. This is a one-way irreversible action — once blacklisted, the code can never be used or un-blacklisted. Requires a valid OTP.

Authentication: SiteOwner

Request Body:

{
  "otp": "String — 6-digit OTP sent to your email (required)"
}

Success Response — 200 OK:

{
  "detail": "Code X7K9-M2P4-QR1W-L5D8 has been blacklisted.",
  "code": "X7K9-M2P4-QR1W-L5D8",

}

Error Responses:

Status Condition
400 Code is already used or already blacklisted
400 Missing OTP
